## Planning Assumptions

- The orchestration engine should stay deterministic and local-first, driven by timers, scripted transitions, and typed events rather than real model calls.
- Coordinator behavior should remain the visible single source of truth for decomposition, synchronization, and completion signaling.
- The existing workspace session model should evolve rather than be replaced, so persistence and later Git isolation can build on the same data contracts.
- Agent communication must be inspectable in the UI as structured messages or events, not hidden behind an opaque runner loop.
- Phase 5 should stop short of real PTY streaming, Git worktree execution, or merge execution; those remain later phases.

## Task List

1. Expand the orchestration domain model.
   - Add shared types for orchestration events, side-chat messages, lifecycle commands, roll-call/status verification, and subtask assignment.
   - Distinguish workspace-facing state from swarm-coordination state without duplicating the whole app model.
   - Ensure persisted app state can store the additional orchestration fields cleanly.

2. Replace the thin mock runner with a believable swarm engine.
   - Refactor `packages/agent-runner` from a per-agent event emitter into a mock orchestration service that can:
     - bootstrap a swarm run
     - emit coordinator decomposition steps
     - emit scout discovery events before builder execution
     - progress builder, reviewer, and tester stages with deterministic state transitions
     - process lifecycle commands such as pause, resume, stop, message-agent, and run-roll-call
   - Keep the engine deterministic enough for tests and demo seed reproducibility.

3. Introduce orchestration reducers/helpers in shared or desktop app state.
   - Add pure helpers that apply orchestration events onto `WorkspaceSession` and related records.
   - Keep event application explicit so app state can be updated incrementally from the mock engine without screen-specific mutation logic.
   - Preserve compatibility with the current persisted workspace session shape.

4. Wire the engine into the desktop app lifecycle.
   - Start the mock swarm run when a workspace session is launched or when the demo session is resumed.
   - Subscribe app state to orchestration events and persist meaningful updates through the existing storage adapter.
   - Add user actions for:
     - pause agent
     - resume agent
     - stop agent
     - message a specific agent
     - trigger coordinator roll call
   - Keep commands local to the active workspace and avoid background behavior that survives app reload in an uncontrolled way.

5. Upgrade the workspace supervision UX around orchestration visibility.
   - Extend the workspace screen to show:
     - structured side-chat or agent-message feed
     - coordinator decomposition and handoff activity
     - roll-call/status verification output
     - clearer per-agent lifecycle controls
   - Make it obvious when scout work precedes builder execution and when reviewer/tester checks gate progress toward merge review.
   - Keep the terminal pane as a mock/log surface unless xterm integration is effectively free.

6. Add tests for orchestration behavior and state shaping.
   - Test deterministic task decomposition and event emission order.
   - Test lifecycle command handling for pause, resume, stop, and direct messaging.
   - Test roll-call/status verification behavior.
   - Test pure event-to-workspace-state reducers/helpers.
   - Preserve lint, typecheck, test, and build cleanliness.

## Constraints And Deviation Rules

- Do not add real model/provider execution in this phase.
- Do not add Git worktree creation or branch manipulation in this phase.
- Do not add xterm.js PTY management unless it is trivial and clearly non-blocking; structured log output is sufficient.
- Do not collapse the UI into a chat-first interaction model.
- Keep the engine inspectable, typed, and small enough to replace later with a real backend.

## Completion Criteria

- ForgeSwarm can run a believable mock swarm lifecycle from structured decomposition through review/test gating.
- The operator can observe inter-agent coordination and roll-call behavior directly in the workspace.
- Agent lifecycle controls and direct messaging work in mock mode.
- Workspace state updates are driven by structured orchestration events rather than only static seeded data.
- The repository remains clean under test, lint, typecheck, and build verification.
